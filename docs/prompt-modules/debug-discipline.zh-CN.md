# Debug 纪律模块

本模块用于错误、失败测试、启动失败、回归和异常行为排查。debug 不允许凭感觉直接改；必须先收集证据，再提出假设，再验证或排除，最后给出最小修复点。

```md
- ...

- ...
```

## 禁止行为

1. 没有证据就直接给根因。
2. 没有复现或证据就直接改代码。
3. 同时改多个不相关点。
4. 用“大概、可能、应该”替代验证。
5. 把症状缓解当成根因修复。

## 正例

- 先复现失败，记录命令和日志，再提出假设表。
- 根因结论包含 confidence 和证据位置。

## 反例

- 看到空指针就直接加判空，不追踪为什么为空。
- 一次提交同时改日志、配置、业务逻辑和测试，无法判断哪个修复生效。
