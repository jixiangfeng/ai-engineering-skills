# 执行纪律模块

本模块用于已经进入实现、修复、迁移落地或 review 选中项修复的任务。目标是按已确认计划执行，不在执行中悄悄扩大范围。

## 执行规则

执行过程中必须遵守：

1. 只改计划内的文件和逻辑。
2. 发现需要扩大 scope 时，必须停下来说明原因。
3. 不顺手重构无关代码。
4. 不修改与目标无关的格式、命名、结构。
5. 每完成一个关键步骤，要更新执行状态。
6. 如果计划不再适用，先修订计划，再继续。

## 输出结构

```md
## Execution Status

| Step | Status | Evidence |
| --- | --- | --- |
| Step 1 | done / pending / blocked | ... |
| Step 2 | done / pending / blocked | ... |
```

## 暂停条件

出现以下情况必须暂停：

- 计划内文件不足以完成目标。
- 需要处理未选择的 finding、未确认的假设或未批准的契约。
- 发现多个可行方案且选择会影响行为、接口、数据或维护成本。
- 当前 diff 出现无关变更或预存变更与本轮改动交叉。

## 正例

- 计划只允许改 service，发现 DTO 也要变时暂停说明并请求确认。
- 每完成一个计划步骤，在执行状态中记录证据。

## 反例

- 顺手格式化整个模块。
- 修 selected finding 时顺便处理未选择 finding。
