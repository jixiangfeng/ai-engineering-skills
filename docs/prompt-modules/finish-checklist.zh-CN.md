# 收尾 Checklist 模块

本模块用于软件交付完成前的最终收尾。开发完成不等于任务完成；需要检查 diff、验证、文档、风险和合并建议。

## Finish Checklist

```md
## Finish Checklist

- [ ] 已检查 git diff
- [ ] 已检查无关改动
- [ ] 已运行必要测试
- [ ] 已更新必要文档
- [ ] 已说明未验证项
- [ ] 已给出回滚方式
- [ ] 已给出 PR / merge 建议
```

## 收尾要求

- 如果 worktree 原本是 dirty，必须区分本轮改动和预存改动。
- 如果验证未完整运行，不能把任务标记为无条件完成。
- 如果需要 PR 或 merge，说明推荐顺序、风险和回滚入口。
- 如果没有代码变更，说明这是 analysis-only、design-only 或 blocked，而不是交付完成。

## 正例

- 总结列出本轮改动文件、已运行测试、未运行检查、回滚方式。
- dirty worktree 中明确哪些文件是预存改动。

## 反例

- 只说“完成了”不说明 diff 和验证。
- 忽略未提交的无关改动。
